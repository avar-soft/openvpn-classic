## 🤔 Зачем это нужно?

Поднять OpenVPN вручную — значит часами разбираться с EasyRSA, TLS-сертификатами, NAT и systemd. Этот скрипт делает всё сам:

- 🔐 **OpenVPN + PKI** — EasyRSA устанавливается и настраивается автоматически, сертификаты генерируются на месте
- 🧑‍💻 **Клиенты** — добавление, отзыв, продление сертификатов через удобное меню
- 📱 **QR-коды** — автоматическая генерация QR для быстрого импорта `.ovpn` на телефон
- 🌐 **Firewall** — iptables / nftables / firewalld настраиваются автоматически, правила восстанавливаются после перезагрузки
- 🔒 **Безопасность** — современные шифры AES-128/256-GCM, ECDSA, TLS 1.2/1.3
- 📋 **Совместимость** — Windows, macOS, iOS, Android, Linux из коробки

## ✨ Возможности

### 🔐 VPN-сервер

| Компонент | Детали |
| --- | --- |
| **Демон** | OpenVPN 2.5+ |
| **PKI** | EasyRSA 3.2.6 |
| **Протоколы** | UDP (рекомендуется) / TCP |
| **Шифрование** | AES-128-GCM, AES-256-GCM, CHACHA20-POLY1305, AES-192-GCM |
| **Сертификаты** | ECDSA (prime256v1 / secp384r1 / secp521r1), RSA (2048 / 3072 / 4096) |
| **TLS** | 1.2 (максимальная совместимость) или 1.3 (современный стандарт) |
| **IPv4/IPv6** | Раздельное управление внутри туннеля |

---

### 🧑‍💻 Управление клиентами

- Добавление клиента с созданием `.ovpn`-профиля и мгновенным QR-кодом
- Отзыв сертификата — немедленная блокировка доступа
- Продление сертификата с настраиваемым сроком действия (до 36500 дней)
- Список всех клиентов с датами и сроками действия сертификатов
- Опциональная защита `.ovpn`-конфига паролем (шифрование закрытого ключа)

---

### 📱 QR-коды

| Вариант | Описание |
| --- | --- |
| **PNG-файл** | Всегда сохраняется рядом с `.ovpn` (`<имя>.qr.png`) |
| **Терминал** | Показывается прямо в консоли, если конфиг < 2.3 КБ |

Передайте PNG на телефон и отсканируйте в **OpenVPN Connect** — никаких ручных настроек.

### 🔥 Firewall и сеть

Скрипт автоматически определяет активный firewall и настраивает нужный:

| Firewall | Поддержка |
| --- | --- |
| **iptables** | ✔ systemd-сервис для восстановления после перезагрузки |

| **firewalld** | ✔ постоянные правила через `firewall-cmd --permanent` |

## 📁 Что создаётся в итоге

```
/etc/openvpn/server/server.conf               # Конфигурация сервера OpenVPN
/etc/openvpn/server/easy-rsa/pki/             # PKI-инфраструктура (CA, сертификаты, ключи)
/etc/openvpn/.menu-clients                    # Реестр клиентских конфигов

/root/<имя>.ovpn                              # Клиентский профиль подключения
/root/<имя>.qr.png                            # QR-код для импорта на телефон (chmod 600)

/etc/sysctl.d/99-openvpn.conf                 # ip_forward и параметры сети
/etc/sysctl.d/99-disable-ipv6.conf            # Отключение IPv6 (если выбрано)

# Firewall (один из вариантов):
/etc/iptables/add-openvpn-rules.sh            # Скрипт добавления правил iptables
/etc/iptables/rm-openvpn-rules.sh             # Скрипт удаления правил iptables
/etc/systemd/system/iptables-openvpn.service  # Автозапуск правил после ребута
/etc/nftables/openvpn.nft                     # Правила nftables (если используется nftables)

/var/log/openvpn-menu/openvpn-install.log     # Лог установки
```

## 📱 Подключение клиентов

После установки скрипт выводит параметры подключения и QR-код. Файл `.ovpn` сохраняется в `/root/<имя>.ovpn`.

### Windows

Скачать **OpenVPN Connect** или **OpenVPN GUI** → открыть `.ovpn`-файл → подключиться.

### macOS

**Tunnelblick** → импортировать `.ovpn`-файл, или **OpenVPN Connect** из App Store.

### iOS / Android

**OpenVPN Connect** → отсканировать QR-код или импортировать `.ovpn`-файл.

### Linux

```bash
sudo apt install openvpn   # или аналог для вашего дистрибутива
sudo openvpn --config <имя>.ovpn
```

## 🔧 Полезные команды после установки

```bash
# Статус службы
systemctl status openvpn-server@server

# Активные VPN-сессии
cat /etc/openvpn/server/openvpn-status.log

# Просмотр логов
journalctl -u openvpn-server@server -n 50

# Перезапуск службы
systemctl restart openvpn-server@server

# Список сертификатов PKI
cat /etc/openvpn/server/easy-rsa/pki/index.txt

# Параметр ip_forward
sysctl net.ipv4.ip_forward

# Правила NAT (iptables)
iptables -t nat -L -n -v
```

## 🖥️ Главное меню

При повторном запуске скрипт автоматически определяет, установлен ли OpenVPN, и показывает нужное меню:

**Если OpenVPN не установлен:**
- Установить OpenVPN — пошаговый мастер
- Установить OpenVPN — интерактивный режим (прямой запуск установщика)
- Показать QR-код для существующего `.ovpn`

**Если OpenVPN установлен:**
- Управление клиентами (добавить / список / QR / отозвать / продлить)
- Управление сервером (статус / продление сертификата / перезапуск / IPv6)
- Переустановить OpenVPN
- Удалить OpenVPN

## 🔐 Безопасность реализации

Скрипт написан с акцентом на надёжность:

- **`umask 077`** — все создаваемые файлы по умолчанию недоступны другим пользователям
- **QR-PNG `chmod 600`** — файл с QR-кодом читает только root
- **Реестр клиентов** — `/etc/openvpn/.menu-clients` отслеживает расположение `.ovpn`-файлов и корректно очищается при удалении
- **Современные шифры** — только AES-GCM и CHACHA20-POLY1305; устаревшие алгоритмы не предлагаются
- **Защита от IPv6-утечек** — опциональное отключение IPv6 через sysctl с предупреждением о NetworkManager
- **Валидация ввода** — все пользовательские значения проверяются перед использованием
- **Trap-очистка** — временный файл встроенного установщика удаляется при любом выходе (`EXIT`, `INT`, `TERM`)
- **Диагностика ошибок** — при сбое установки автоматически выводятся последние строки лога с подсказками
